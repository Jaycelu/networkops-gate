## **1. 概述**

### **1.1 背景与目的**
随着企业数字化转型加速，网络规模扩大，传统手动配置网络设备的方式效率低下且易出错。网络工程师面临架构设计、IP规划、设备配置、批量管理和配置校验等多重挑战，中小企业尤其缺乏专业人才支持。现有工具功能分散，缺乏集成性，用户体验不佳。

本产品分为两个主要组成部分：
1. **Web官网**：基于HTML/CSS/JavaScript构建的营销官网，提供产品介绍、用户注册登录、账单管理、应用下载等功能
2. **移动桌面端应用**：基于AI大模型驱动的专业工具，支持Windows、macOS、Linux平台，帮助用户快速完成网络架构设计、设备配置及管理

产品将支持大模型选择，结合灵活的收费模式（基于token的AI功能收费+普通功能一次性买断），通过规则校验模块提升AI输出准确性，满足不同用户需求。

### **1.2 产品愿景**
成为网络工程师和企业IT团队的首选工具，通过AI驱动的自动化和多端集成服务，简化网络设计与管理流程，助力企业高效完成数字化转型，并通过灵活的商业模式实现可持续盈利。

### **1.3 目标用户**
- **主要用户**：网络工程师、IT运维团队、网络集成商。
- **次要用户**：中小企业IT负责人、云服务提供商技术团队。
- **用户画像**：
  - 年龄：25-45岁
  - 职业背景：具备一定网络技术基础，但部分用户可能缺乏高级网络设计经验。
  - 痛点：网络设计复杂、配置耗时、批量操作易错、缺乏统一工具、设备环境多样化、AI输出不准确。

### **1.4 商业目标**
- **短期（6-12个月）**：推出MVP版本，验证核心功能，获取首批用户反馈，目标覆盖10-20家付费客户。
- **中期（12-24个月）**：完善功能，支持多厂商设备，扩展用户群体，目标覆盖100+中小企业客户。
- **长期（24个月以上）**：成为行业领先的网络自动化工具，支持云原生网络和SDN，探索订阅制、买断制和token计费的多模式盈利。

## **2. 产品范围与优先级**

### **2.1 MVP（最小可行产品）范围**

#### **Web官网功能**
1. **产品介绍页面**：展示核心功能、价值主张、技术特色
2. **用户注册登录**：支持邮箱注册、密码登录、用户信息管理
3. **账单管理**：Token余额查看、使用记录、充值功能
4. **应用下载**：支持Windows、macOS、Linux版本下载
5. **联系我们**：提供多种联系方式和反馈渠道

#### **移动桌面端应用功能**
1. **架构设计与IP规划**：根据用户输入生成网络架构和IP地址分配方案（AI驱动）。
2. **设备配置生成**：基于设计结果生成主流设备（如Cisco交换机）的配置命令（AI驱动）。
3. **规则校验模块**：对AI生成的网络设计和配置脚本进行规则校验，确保符合网络工程最佳实践和厂商标准（初期支持Cisco、Huawei和H3C）。
4. **批量配置与修改**：通过SSH批量推送配置到设备（非AI功能）。
5. **配置备份与导入导出**：支持设备配置的备份、导入和导出功能（非AI功能）。
6. **基线文档生成**：根据设计方案自动生成网络基线文档（AI驱动）。
7. **大模型选择**：系统配置中支持用户选择大模型API代理（如aihubmix）。

### **2.2 后续迭代功能（非MVP）**

#### **Web官网扩展**
1. **用户社区**：技术博客、用户案例分享、问题讨论区
2. **在线文档**：详细使用指南、API文档、技术支持
3. **企业服务**：定制化服务咨询、合作伙伴招募

#### **移动桌面端应用扩展**
1. **配置检查与合规性校验**：AI检查配置是否符合客户基线，并提供修复建议。
2. **多厂商支持扩展**：扩展对Juniper等设备的支持。
3. **可视化拓扑图**：以图形化方式展示网络架构。
4. **日志与审计**：记录配置变更操作，支持问题追溯。

## **4. 系统架构与技术要求**

### **4.1 系统架构概览**

#### **4.1.1 Web官网架构**
- **技术栈**：HTML5 + CSS3 + JavaScript (ES6+)
- **UI框架**：Tailwind CSS + Iconify图标库
- **交互功能**：原生JavaScript实现注册登录、账单管理等
- **部署**：阿里云ECS + CDN静态资源加速

#### **4.1.2 移动桌面端应用架构**
- **前端**：
  - **跨平台框架**：基于Electron框架开发，支持Windows、macOS、Linux
  - **UI界面**：采用现代化的桌面应用设计，与原型图保持一致
- **后端**：
  - API服务基于Flask框架，处理用户请求。
  - AI模块集成大模型API代理（如aihubmix），支持动态切换大模型。
  - 规则校验模块：独立服务，基于Python开发，校验AI输出（详见4.2）。
  - 自动化脚本引擎基于Python Paramiko，用于设备交互（如SSH配置推送）。
- **数据存储**：
  - 本地SQLite数据库存储用户信息、设计方案、配置历史
  - 云端同步支持（可选）
- **设备交互**：支持SSH/串口通信。
- **部署**：通过Electron打包为各平台安装包进行分发

### **4.2 规则校验模块详细设计**
#### **4.2.1 模块目标**
- 提高AI生成结果的准确性，确保网络架构设计、IP规划和设备配置符合网络工程最佳实践和厂商标准（初期支持Cisco、Huawei和H3C）。
- 增强用户信任，提供校验报告和修改建议。

#### **4.2.2 模块逻辑框架**
1. **输入解析**：
   - 从AI输出中提取结构化数据（如JSON格式的网络设计、文本格式的配置脚本）。
   - 使用正则表达式或简单NLP工具提取关键字段（如IP地址、VLAN ID、命令关键词）。

2. **规则匹配与校验**：
   - 基于预定义规则库，检查AI输出是否符合标准。
   - 规则库分类：
     a. 网络架构设计与IP规划规则：
        - IP地址合法性：检查IP地址和子网掩码是否有效。
        - 子网划分合理性：确保子网无重叠，在指定范围内。
        - VLAN数量与ID范围：ID在1-4094，数量合理。
        - 设备数量与架构一致性：冗余性要求下检查设备数量。
     b. 设备配置脚本规则（Cisco、Huawei和H3C）：
        - 语法正确性：检查命令是否符合厂商语法。
        - 上下文依赖性：检查命令执行顺序。
        - 安全合规性：检查是否存在安全隐患（如未禁用Telnet）。
        - 最佳实践一致性：检查是否遵循常见实践。

3. **输出处理**：
   - 自动修正：对可明确修正的错误（如非法子网掩码），系统自动调整。
   - 提示用户：对不可修正问题，生成错误/警告报告，提示修改建议。
   - 通过校验：直接输出结果供用户使用。
   - 输出校验报告（JSON/HTML格式），包含错误、警告和修正项。

#### **4.2.3 规则库设计（初期支持Cisco、Huawei和H3C）**
- **网络架构设计与IP规划规则**：
  | 规则名称 | 校验逻辑 | 优先级 | 错误处理方式 |
  |----------|----------|--------|--------------|
  | IP地址合法性 | 检查IP地址格式和掩码范围（/0-/32） | 高 | 自动修正或提示 |
  | 子网划分合理性 | 检查子网无重叠，在范围内，主机数合理 | 高 | 自动修正（如掩码）|
  | VLAN数量与ID范围 | VLAN ID在1-4094，数量不超过设备上限 | 中 | 提示减少数量 |
  | 设备数量与架构一致性 | 冗余要求下检查是否有足够设备或路径 | 中 | 提示补充设备 |

- **Cisco设备配置规则**：
  | 规则名称 | 校验逻辑 | 优先级 | 错误处理方式 |
  |----------|----------|--------|--------------|
  | 命令语法正确性 | 检查是否符合Cisco IOS语法 | 高 | 提示调整命令 |
  | 接口模式依赖性 | 检查配置前是否进入接口模式 | 高 | 自动插入前置命令 |
  | VLAN接口启用 | 检查是否配置`no shutdown` | 中 | 自动添加命令 |

## **5. 商业模式与收费设计**

### **5.1 收费模式概述**
产品采用混合收费模式，区分AI功能和非AI功能，满足不同用户需求。
1. **AI功能收费**：基于token使用量计费，适用于架构设计、设备配置生成、配置检查等功能。
2. **非AI功能收费**：一次性买断付费，适用于批量配置与修改、配置备份等功能。
3. **Web官网服务**：基础功能免费，增值服务收费。

### **5.2 具体收费设计**

#### **5.2.1 AI功能（按token使用量计费）**
- **计费单位**：每1000 token收费$1（具体根据API代理成本调整）。
- **预充值**：用户需购买token包（如$10起充，包含10000 token）。
- **使用监控**：系统记录每次AI调用的token消耗，提供余额查询和充值提醒。
- **免费试用**：新用户赠送少量token（如1000 token），体验AI功能。

#### **5.2.2 非AI功能（一次性买断付费）**
- **基础版**：一次性支付$200，永久使用批量配置与修改、配置备份等功能。
- **扩展包**：支持设备数量上限（如50台），超出需购买扩展包（如$50加10台设备）。

#### **5.2.3 Web官网增值服务**
- **高级支持**：$100/年，提供优先技术支持和问题解答
- **企业服务**：按需定制，包括私有部署、定制功能开发等

### **5.3 支付与账单管理**
- **支付方式**：支持信用卡、支付宝、微信支付，集成第三方支付平台（如Stripe、支付宝SDK）。
- **账单系统**：
  - 用户可查看token使用明细和充值记录。
  - 买断版用户可查看购买历史和扩展包状态。
  - 提供电子发票功能，满足企业用户需求。

## **6. 用户流程图**

### **6.1 完整用户流程**

#### **6.1.1 新用户首次使用流程**
用户访问官网 → 浏览产品介绍 → 注册账号（获得免费Token） → 下载桌面应用 → 安装应用 → 配置大模型API → 开始使用AI功能

#### **6.1.2 核心业务流程**
登录桌面应用 → 选择"新建设计" → 输入网络参数 → AI生成设计方案（token计费） → 规则校验（校验报告+修正/提示） → 用户确认/调整 → 生成配置脚本（token计费） → 批量推送配置到设备（买断付费） → 返回执行结果

#### **6.1.3 账单管理流程**
用户登录官网 → 进入账单页面 → 查看Token余额 → 查看使用记录 → 充值Token（可选） → 查看买断功能状态 → 导出账单（可选）

### **6.2 页面跳转与交互流程**

#### **6.2.1 Web官网交互流程**
1. **首页导航流程**：
   - 用户访问官网，页面以淡入动画方式加载
   - 顶部导航栏显示产品Logo和核心功能入口
   - 功能卡片区域展示产品特色，鼠标悬停有视觉反馈
   - 点击下载按钮跳转到下载页面

2. **用户账户流程**：
   - 点击注册/登录按钮弹出模态框
   - 注册成功后跳转账单页面
   - 登录后导航栏显示用户菜单

3. **下载流程**：
   - 选择对应平台版本
   - 登录验证后开始下载
   - 显示下载进度和完成提示

#### **6.2.2 桌面应用交互流程**
1. **应用启动流程**：
   - 显示加载动画
   - 检查用户登录状态
   - 加载本地数据和配置

2. **设计生成流程**：
   - 用户填写网络参数表单
   - 点击生成按钮显示加载状态
   - AI处理完成后展示结果
   - 规则校验自动执行

3. **配置部署流程**：
   - 用户确认设计方案
   - 生成设备配置脚本
   - 输入设备连接信息
   - 执行批量部署并显示进度

### **6.3 用户操作状态与权限管理**

#### **6.3.1 Token管理**
- 桌面应用实时显示Token余额
- 余额不足时AI功能按钮变灰并提示充值
- 官网提供Token充值入口和历史记录

#### **6.3.2 功能访问控制**
- 未配置大模型API时，AI功能不可用
- 买断功能未购买时，相关功能受限
- 系统根据用户权限动态调整界面

## **8. 风险与缓解措施**

| 风险类型 | 描述 | 影响程度 | 缓解措施 |
|----------|------|----------|----------|
| 技术风险 | AI生成的设计或配置可能不准确 | 高 | 增加规则校验模块和用户确认环节 |
| 兼容性风险 | 不同厂商设备配置语法差异大 | 中 | 优先支持主流厂商，逐步扩展支持范围 |
| 安全风险 | 设备凭据或API密钥泄露 | 高 | 加密存储凭据，使用系统密钥库 |
| 用户接受度风险 | 用户对token计费模式不熟悉 | 中 | 提供透明计费说明和免费试用token |
| 部署风险 | 跨平台打包和分发复杂 | 中 | 使用成熟的Electron生态，充分测试各平台 |
| 校验规则不足风险 | 规则校验模块覆盖场景有限 | 中 | 收集用户反馈，动态更新规则库 |
| 设备支持风险 | H3C等新厂商设备支持可能存在兼容性问题 | 低 | 建立设备兼容性测试机制，逐步完善支持 |
| 网络依赖风险 | API调用依赖网络连接 | 中 | 提供离线模式和网络重试机制 |
